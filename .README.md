# AI-Powered TODO Task Manager CLI

A command-line interface (CLI) application that helps users manage complex tasks by leveraging artificial intelligence to break down and destructure complex task descriptions into actionable subtasks.

## 📚 Academic Context

This project is developed as part of a **Master's course** on implementing AI tools in software development. It serves as a hands-on exploration of integrating AI capabilities into everyday software solutions.

## 🎯 Project Goals

- **Task Management**: Create a simple, intuitive CLI for managing todo tasks
- **AI Integration**: Use AI to intelligently parse and destructure complex task descriptions
- **Persistence**: Store tasks locally in JSON format (no database required)
- **Clean Architecture**: Build on object-oriented principles with clear separation of concerns

## 🌟 Features

### Planned Features for v0.1 (Basic Functionality)

- CLI menu interface with intuitive navigation
- Task model with properties: id, description, completed
- TaskManager class for CRUD operations
- JSON-based persistence layer
- View, create, update, and delete tasks

### Planned Features for v1.0 (AI Integration)

- **AI Task Destructuring**: Convert complex task descriptions into organized subtasks
- 💾 **Auto-save**: Automatically persist changes to JSON

### Core Components

#### Task Class (`task.py`)

Represents a single todo task with:

- `id`: Unique identifier
- `description`: Detailed task description
- `completed`: Boolean status
  Handles completed status with:
- `mark_completed()`: Mark task as completed
- `is_completed()`: Check if task is completed
  Overrides `__str__()` for user-friendly display of task information.

#### TaskManager Class (`task_manager.py`)

Handles all task operations:

- `create_task()`: Add a new task
- `complete_task()`: Mark task as completed
- `delete_task()`: Remove task
- `list_tasks()`: Get all tasks

#### Persistence Layer (`persistence.py`)

Manages JSON file operations:

- Load tasks from `tasks.json`
- Save tasks to `tasks.json`
- Handle serialization/deserialization

#### CLI Menu (`main.py`)

User interface:

- Main menu loop
- Task creation workflow
- Task completion/deletion tracking
- Exit handling

#### Constants Module (`constants.py`)

Centralized constants for the application:

- **Menu options**: Menu choice labels and descriptions
- **Displayed text**: User prompts, success/error messages
- **Messages**: Info, warning, and error messages for user feedback
- **UI formatting**: Separators, titles, and formatting characters

#### Configuration Module (`config.py`)

Task properties and application settings:

- **Task attributes**: Default task properties and field definitions
- **Task schema**: Field names, types, and validation rules
- **File paths**: Configurable paths for data storage and logs

#### Environment Variables (`.env-example`)

Template for environment configuration. Copy this file to `.env` and fill in your values:

**Important**: `.env` should never be committed to version control. It's listed in `.gitignore`.

## 🚀 Getting Started

### Prerequisites

- Python 3.8+
- pip or conda for package management

### Installation

1. **Clone the repository**

   ```bash
   cd ai-to-do-list
   ```

2. **Create a virtual environment**

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   ```

3. **Install dependencies** Not yet finalized, but will include AI libraries in the future:
   ```bash
   pip install -r requirements.txt
   ```

### Running the Application

Start the CLI application:

```bash
python main.py
```
